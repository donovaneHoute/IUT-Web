1. Introduction

Une grille (https://www.w3.org/TR/css-grid/) permet d’organiser les éléments sur deux axes (lignes et colonnes).

Tout comme pour la méthode des boites flexibles, c’est la propriété display qui permet de transformer un élément en conteneur de grille.

Pour les grilles, le conteneur doit donc posséder la déclaration display:grid (ou inline-grid si le conteneur doit être un élément en bloc).

Les propriétés principales pour définir la grille sont :

grid-template-columns, grid-template-rows

Définissent les colonnes et les lignes.

gap

Définit l’espace entre les cellules.

grid-column, grid-row

Positionnent les éléments selon les numéros de lignes.

grid-template-areas

Permet de nommer les zones de la grille.

1.1. Historique

La première normalisation de cette méthode date d’avril 2011, mais c’est en 2017 que les grands navigateurs l’ont pris charge (L’histoire des grilles par ses créateurs).

La version 2, démarrée en février 2018, a pour objectif principal l’ajout de la valeur subgrid (dernière mise à jour mars 2025). Cela permet à une grille imbriquée (un élément enfant qui est lui-même display:grid) de participer aux mêmes lignes de dimensionnement que la grille parent. Cela permet d’aligner plus finement les grilles imbriquées. https://developer.mozilla.org/fr/docs/Web/CSS/Guides/Grid_layout/Subgrid

La première description officielle de la version 3 date de septembre 2024. Elle ajoute une nouvelle méthode d’empilement des boites (masonry) pour répondre aux nouvelles exigences de mise en page. Cette version est encore en phase d’étude, donc il est déconseillé de l’utiliser en production pour le moment. Nous ne l’utiliserons pas dans ce TP.

Avec les méthodes de positionnement des boites en grille ou flexible, il est possible de changer le positionnement et le comportement d’une seule boite enfant en utilisant des propriétés spécifiques ciblant directement cette boite. Ces propriétés surchargeront alors le comportement global déclaré dans le conteneur.

1.2. Un peu de vocabulaire

Vocabulaire
Grille, grid

La zone complète définit par le conteneur.
Cellules, cells

Les cases de la grille.
Lignes, lines

Elles divisent l’espace de la grille en cellules. Elles sont numérotées à partir de 1 en haut à gauche, ou à partir de -1 en partant d’en bas à droite.
Piste, track

La zone entre deux lignes de la grille, sur toute sa longueur (= ce que vous risquez d’appeler ligne ou colonne par erreur).
Gouttière, gutter

C’est l’espacement entre deux pistes. Son épaisseur est définie par la propriété `gap`[1].
Item, Item

Un élément enfant du conteneur.
Zone, area

Surface couverte par un item dans la grille. Les zones peuvent être nommées pour en simplifier le positionnement.
Figure 1. Vocabulaire en image
¡ Attention !

Ce sont les lignes qui sont numérotées, pas les pistes.
On dit qu’une grille est définie :

  • explicitement si elle est totalement définie dans le conteneur avec les propriétés grid-template[-xxx] où xxx vaut rows, columns ou areas.

  • implicitement si elle est définie partiellement. Les pistes manquantes (selon le nombre d’items et leur position souhaitée) seront ajoutées automatiquement en suivant le comportement demandé, localement (propriété dans un item) ou globalement (dans le conteneur).

On peut définir des comportements pour des placements entièrement implicites avec les propriétés grid-auto-xxx où xxx vaut rows, columns ou flow.

1.3. Nouvelle unité : fr

Une nouvelle unité est apparue avec les grilles : la fraction (fr). Elle permet de répartir la taille des pistes de manière proportionnelles entre elles. Cela évite d’avoir à calculer soi-même les tailles de chaque piste.

La répartition se fait uniquement pour les espaces non contraints (i.e. : pistes de taille non fixes).

Par exemple, grid-template-columns: 1fr 20em 2fr; permet de créer 3 colonnes dont la seconde est de taille fixe et la 3e colonne est le double de la première sur l’espace modulable.

Pour un nombre de pistes identiques et consécutives, on pourra avantageusement utiliser la fonction repeat.

Exemple avec la grille de la Figure 1, “Vocabulaire en image”
grid: repeat(5, 1fr) / repeat(4, 1fr);

La propriété grid est une propriété raccourcie qui permet de définir toutes les propriétés grid-template-xxx ou grid-auto-xxx.

Il existe d’autres fonctions et valeurs intéressantes (minmax(), max-content, min-content ou encore autofill et auto-fit à utiliser avec la fonction repeat()). Renseignez-vous sur ces valeurs associées.

1.4. Création du quadrillage dans le conteneur

Par défaut les items sont ajoutés dans leur ordre d’apparition dans le flux.

Deux méthodes de création de grilles sont possibles.

1.4.1. Méthode 1 : Numérotation des lignes

C’est la méthode la plus courante.

Elle utilise la numérotation des gouttières pour indiquer le début (grid-column-start, grid-row-start) et la fin (grid-column-end, grid-row-end) d’un item.

  • grid-columns et grid-rows sont des propriétés raccourcies. Elles s’écrivent

    grid-columns: grid-column-start / grid-column-end;
grid-rows: grid-row-start / grid-row-end;

  • grid-area est la propriété raccourcie pour l’ensemble de ces informations. Elle s’écrit alors :

    grid-area: grid-row-start / grid-column-start / grid-row-end / grid-column-end;
Exemple : Zone verte de la Figure 1, “Vocabulaire en image”, cela donne :
.green_area {
  grid-column-start: 4;             | /* ou  par valeur d'"étalement" */
  grid-column-end: 6;               |   grid-column : 4 / span 2;
  grid-row-start: 1;                |   grid-row : 1 / span 2;
  grid-row-end: 3;                  |
                                    | /* ou encore en 1 ligne : */
/* ou */                            |   grid-area : 1 / 4 / 3 / 6;
  grid-column : 4 / 6;              | /* ou même : */
  grid-row : 1 / 3;                 |   grid-area : 1 / 4 / span 2 / span 2;
}
/* span permet de définir une longueur en nombre de pistes à couvrir.*/

1.4.2. Méthode 2 : Nommage des zones

Pour nommer des items, il faut utiliser la propriété grid-area sur chacun d’eux.

La disposition des items sera alors décrite dans le conteneur avec la propriété grid-template-areas. Il faut nommer chaque cellule d’une zone avec le nom de l’item qui va s’y trouver.

Quelques contraintes pour la définition de la grille :

  • Les mots span et auto sont interdits comme nom de cellule.

  • Les cellules de même nom doivent former une zone (au sens de la grille) et une seule.

  • Toutes les cellules explicitement définies doivent avoir :

    • soit un nom (comme défini par la propriété grid-area),

    • soit le caractère point ., indiquant une cellule vide.

Exemple

Avec les deux premières pistes horizontales de la figure 1, et en supposant que les deux cellules bleues ne forme qu’un item ; Le positionnement se fait dans le container, le nommage est défini dans les items. Cela donne ceci :

.container {                           ┆.item-vert {                   ┆.item-bleu1 {
  display: grid;                       ┆   grid-area: area;            ┆  grid-area: cell1;
  grid-template-areas :                ┆   background-color: green;    ┆  background-color: blue;
      "cell1 cell2 track area area"    ┆}                              ┆}
      "  .     .   track area area";
}

.item-rouge {                          ┆.item-bleu2 {
  grid-area: track;                    ┆  grid-area: cell2;
  background-color: red;               ┆  background-color: blue;
}                                      ┆}

Un très bon exemple responsive (= utilisant des règles @media) est accessible dans la documentation officielle, n’hésitez pas à aller y jeter un œil, c’est par ici : https://drafts.csswg.org/css-grid/#source-independence
